The ASIC Inversion

A proof-of-work algorithm where CPU architecture becomes the load-bearing computation itself. ASICs that attempted to optimize for Stratum would, by definition, become CPUs.

100,000× Traditional ASIC advantage
47× Stratum efficiency gap

Problem

SHA-256 and similar algorithms strip CPUs to their core operations—bitwise functions and modular arithmetic. Every other CPU feature (caches, branch prediction, speculative execution) becomes wasted overhead. ASICs eliminate this waste, achieving efficiency ratios as high as 100,000×. This creates insurmountable barriers to CPU mining.

Solution

Stratum makes CPU overhead the computation. Multi-level cache hierarchies, branch prediction, and speculative execution are not overhead—they are the algorithm. This creates structural parity between commodity hardware and any hypothetical specialized circuit.

Core Components

ZK Proofs of Cache

STARK proofs verify cache execution. Verification: O(log n) time, ~50ms. Proof size: ~120 KB. No trusted setup required.

Software Cache Model

Canonical cache geometry: L1 (256 KB), L2 (1 MB), L3 (12 MB). Deterministic behavior across all hardware. 18–22% overhead vs. native execution.

ISA Profiles

Algorithm specified as core logic plus ISA profiles. Portable across processor architectures while maintaining security guarantees.

Hardware Parity

An ASIC that replicates Stratum's cache behavior is, structurally, a CPU. Commodity hardware becomes competitively viable.
